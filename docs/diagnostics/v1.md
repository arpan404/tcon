# Diagnostics Codes v1

This document defines stable machine-readable error codes emitted when running:

- `tcon --error-format json ...`

## JSON schema

Errors are emitted on stderr as a single JSON object:

```json
{
  "code": "E_RUNTIME",
  "message": "human-readable message",
  "file": "/abs/or/relative/path.tcon",
  "line": 12,
  "col": 8
}
```

`file`, `line`, and `col` may be `null` if location extraction is unavailable.

## Stable codes

- `E_LEX_UNEXPECTED_CHAR` - lexer encountered an unsupported character.
- `E_LEX_UNTERMINATED_STRING` - unterminated string literal.
- `E_LEX_UNTERMINATED_BLOCK_COMMENT` - block comment (`/* ... */`) without closing `*/`.
- `E_EVAL_MISSING_EXPORT` - required export (`spec`, `schema`, `config`) missing.
- `E_IMPORT_CYCLE` - circular import detected.
- `E_VALIDATE_ENUM` - enum value not in allowed variants.
- `E_VALIDATE_STRICT_UNKNOWN_KEY` - `config` included keys not declared on a `.strict()` object.
- `E_VALIDATE_SECRET` - secret-field policy violation (`.secret()` usage, hardcoded secret value, or secret default).
- `E_SPEC_UNKNOWN_KEY` - `spec` object contained an unrecognized field name.
- `E_SPEC_FORMAT` - unsupported `spec.format`.
- `E_SPEC_PATH_ESCAPE` - output path escapes workspace root after canonical/symlink resolution.
- `E_OUTPUT_COLLISION` - two entries emit to the same output path.
- `E_ENV_INTERPOLATION` - environment interpolation failure (unset variable / malformed interpolation).
- `E_EMIT_NULL_UNSUPPORTED` - chosen emitter cannot represent `null` in the normalized value.
- `E_PARSE_OR_SCHEMA` - parser/schema construction failure.
- `E_RUNTIME` - generic fallback.

## Compatibility promise

For v1, `code` values above are stable identifiers and should be used by integrations for automation and alerting.
